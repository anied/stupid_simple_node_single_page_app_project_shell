Any path will redirect back to this HTML file (set as "approot" in the config/settings.json file in this project). Check it out:

/somePath

some/other/path

If you were utilizing an MV* library like Backbone, AngularJS, React, Ember, or whatever, you could capture those routes and display the proper content, regardless of where the user entered from-- if they bookmarked some/specific/application/route, they would still get the approot, which could then load up and handle the route appropriately.

For more information, checkout the project on Github.
